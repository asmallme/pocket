我想创建一个以分享自己的收藏内容为主的网站或者App。可以将自己日常浏览各种网站或者app中有价值的内容收藏起来，一是方便自己稍后阅读，另一方面可以让别人阅读，同时阅读别人的可以扩展阅读面。
基于这样一个想法，我需要实现一个可以收藏内容的网站或者App。
1. 用户可以收藏内容，收藏的内容可以有文字、图片、视频等形式。
2. 用户可以分享自己的收藏内容，分享的内容可以有文字、图片、视频等形式。
3. 用户可以阅读别人的收藏内容，阅读别人的收藏内容可以扩展阅读面。
4. 用户可以评论别人的收藏内容，评论别人的收藏内容可以表达自己的观点。
5. 用户可以点赞别人的收藏内容，点赞别人的收藏内容可以表达自己的喜欢。
6. 可以订阅别人的收藏内容。默认展示全局的收藏feed流。

对于收藏内容的方式需要做到极致的方便快捷，用户可以方便的收藏内容，并且可以方便的分享内容。可以来自app或者网页，因此需要实现一个插件或者快捷方式等。

基于这些想法和诉求，我们先展开讨论一下这个项目如何实现，以及一些关键的产品设计方案和技术方案。

## v1.1 实施计划（细化）

### 1. 右键唤醒（细分为四个入口）
- 收藏当前页面：注入脚本提取页面元数据后直接保存
- 收藏选中文字：划词收藏，选中内容作为推荐语、当前页作为来源链接
- 收藏这张图片：图片类型收藏，图片地址作为封面
- 收藏这个链接：收藏页面里的超链接，调用 unfurl 接口补全标题
- 附加：快捷键一键收藏（Cmd/Ctrl+Shift+S）、角标即时反馈、失败暂存重试队列

### 2. 特定网站抓取优化（站点适配器）
- 抓取重心放在插件 content script（服务端抓不到登录墙后的内容）
- 按域名匹配的适配器注册表，第一批：X（单条帖子正文/作者/配图）、
  微信公众号文章、知乎问答、B站视频
- 未命中适配器时回退到通用 OG 标签提取

### 3. 插件与网页登录态共享
- 插件登录页增加「使用网页账号登录」，跳转网页 /extension-auth 授权页
- 网页端已登录则一键授权：服务端用 service role 换发一次性 token，
  通过浏览器消息通道交给插件，插件用它建立自己独立的会话
- 两端会话相互独立，避免 refresh token 轮换互踢的问题

### 4. 网页个人中心增强
- 头像上传（存储到 media 桶）
- 修改密码
- 账号信息展示：邮箱、注册时间
- 修改邮箱依赖 SMTP 配置，推迟到配好邮件服务之后

### 5. 数据与体验补充
- bookmarks 表增加 source 列（web/extension/contextmenu/shortcut/pwa-share），
  记录收藏入口便于后续分析
- 重复收藏检测：同一 URL 再次收藏时提示已收藏过
- 实施顺序：1 → 2 → 3 → 4
